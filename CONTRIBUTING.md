# Contributing to Catryna Wikinelli 🐱

Thank you for your interest in contributing! Every contribution helps make Catryna purr louder.

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Install** dependencies with `bun install`
4. **Create** a feature branch from `main`

## Development

```bash
# Start development servers
bun run dev

# Run tests
bun run test

# Type check
bun run typecheck

# Lint
bun run lint
```

## Code Style

- We use **TypeScript** everywhere
- **Prettier** for formatting (runs on save)
- **ESLint** for linting
- Prefer **functional components** with hooks
- Use **descriptive variable names** over comments

## Commit Messages

We follow conventional commits:

```
feat: add new block type for embeds
fix: resolve search indexing issue
docs: update MCP integration guide
refactor: simplify coverage calculation
```

## Pull Requests

1. Update documentation if needed
2. Add tests for new features
3. Ensure all checks pass
4. Request review from maintainers

## Reporting Issues

- Use the issue template
- Include reproduction steps
- Attach relevant logs or screenshots

## Questions?

Open a discussion or reach out to the maintainers.
